• Sistema de control de versiones
    1. Locales : Se almacena en una base de datos, en cualquier momento solo se tiene una copia del proyecto, evitando la confusión o eliminacion de versiones
    2. Centralizados : Se almacena en un servidor, se impide que los usuarios invaliden el trabajo de los demas, cada conflicto debe de solucionarlo alguien de forma manual
    3. Distribuidos : Cada desarrollador tiene una copia local de todo el proyecto, se evitan conflictos y posibles caidas del servidor
